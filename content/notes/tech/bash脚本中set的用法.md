`set` 命令是Bash脚本的核心命令，用于**修改Shell环境的运行参数**，定制脚本执行环境，提升脚本的安全性、可维护性和调试效率；无参数直接执行`set`时，会显示当前Shell所有的环境变量和Shell函数。

网页中重点介绍了**4个最常用的参数**，包含简写和完整写法，各参数可组合使用，以下是具体参数说明及用法：

## 1. `-u / -o nounset`  未定义变量

检测未定义的变量，遇到**不存在/未绑定的变量**时，脚本直接报错并终止执行，避免Bash默认忽略未定义变量导致的隐性问题。

- **Bash默认行为**：遇到未定义变量时，仅输出空行，脚本继续执行。
- **适用场景**：强制校验脚本中变量的定义和使用，减少变量拼写错误、未初始化带来的bug。

## 2. `-x / -o xtrace` 调试模式

开启脚本**调试模式**，执行脚本时，会在**运行结果前先输出执行的命令行**，命令行首以`+`标识，清晰展示脚本的执行流程。

- **Bash默认行为**：仅输出命令运行结果，不显示执行的命令本身，复杂脚本难以定位输出对应的命令。
- **适用场景**：调试复杂Bash脚本，快速定位执行异常的命令行。

## 3. `-e / -o errexit` 错误终止

开启**错误终止模式**，脚本中**任意命令执行失败（返回值非0）** 时，立即终止脚本执行，防止错误累积。

- **Bash默认行为**：命令执行失败仅输出错误信息，脚本继续执行后续命令，易导致后续操作基于错误结果执行，引发连锁问题。
- **特殊处理**：若需某个命令失败后脚本仍继续执行，有两种方式：
  1. 临时关闭：`+e`（关闭）→ 执行目标命令 → `-e`（重新开启）；
  2. 命令后追加：`command || true`（让该行语句始终执行成功）。
- **局限性**：对**管道命令**失效，管道命令的返回值默认取最后一个子命令的返回值，只要最后一个子命令成功，整个管道命令即判定为成功。

**ERR继承**： `-E / -o errtrace` ，只要脚本中的任何地方（包括深层嵌套的函数或子进程）发生错误，都会触发全局定义的 ERR 陷阱。

## 4. `-o pipefail` 管道失败

- **作用**：弥补`-e`的管道命令缺陷，**只要管道命令中有一个子命令执行失败**，整个管道命令的返回值即为失败（非0），结合`-e`可让脚本在管道命令出现错误时立即终止。
- **Bash默认行为**：管道命令的返回值仅由最后一个子命令决定，前面子命令失败不影响整体判定。
- **使用要求**：必须与`-e`配合使用，单独使用无实际错误终止效果。

### 5. 常用组合及使用方式

上述4个参数是Bash脚本的**标配参数**，建议放在所有Bash脚本的头部，也可在执行脚本时通过命令行传入，两种核心使用方式：

方式1：脚本内直接组合设置

```bash
# 写法1：一行组合所有参数
-euxo pipefail

# 写法2：拆分设置（效果一致）
-eux
-o pipefail
```

方式2：执行脚本时命令行传入

无需修改脚本，执行时通过`bash`命令附带参数：

```bash
bash -euxo pipefail 脚本名.sh
```

`set`命令的这4个参数从**变量校验、调试追踪、错误终止、管道校验**四个维度，解决了Bash脚本默认执行模式下的安全性和可维护性问题，是编写健壮、可靠Bash脚本的基础配置。
