前端环境与依赖管理工具：

  • Node 是运行环境，是所有工具的基础；
  • npm 是 Node 自带的包管理器，用来装依赖；
  • nvm/n/Volta 是管 Node 版本的（比如项目 A 用 Node 18，项目 B 用 Node 20）；
  • pnpm 是管依赖安装的（比 npm 更快、更省空间，可直接替换 npm）。
  • nrm 管理 npm 源：快速切换淘宝源、官方源、公司私有源

node 版本管理

使用 nvm（以 Windows/macOS 为例）

1. 安装
# 安装 nvm（macOS/Linux）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
# 或用 wget
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 验证安装（重启终端后）
nvm --version

# （2）安装（Windows）
# 下载 nvm-windows 安装包：https://github.com/coreybutler/nvm-windows/releases，安装后验证：
nvm version
2. 核心命令（通用）
# 1. 查看可安装的 Node 版本
nvm ls-remote  # 查看所有远程版本
nvm ls-remote --lts  # 仅查看 LTS（长期支持）版本

# 2. 安装指定版本
nvm install 18.18.0  # 安装 Node 18.18.0
nvm install lts  # 安装最新 LTS 版本

# 3. 查看已安装版本
nvm ls

# 4. 切换版本（全局生效）
nvm use 18.18.0

# 5. 设置默认版本（重启终端仍生效）
nvm alias default 18.18.0

# 6. 卸载指定版本
nvm uninstall 14.21.0

极简方案：n（npm 全局安装）

# 安装 n
npm install -g n

# 安装指定版本
n 18.18.0
n lts  # 安装 LTS 版本
n latest  # 安装最新版本

# 切换版本（需管理员权限）
n  # 交互式选择已安装版本

Linux 系统中，n 切换版本，是通过 copy 对应版本的 node 到 /usr/local/bin/node* 来实现的。

volta

项目依赖管理

npm, pnpm

1. 依赖类型（必须分清）

类型作用安装命令
dependencies生产环境依赖（项目运行必需，如 React、Vue、axios）npm install axios（默认）
devDependencies开发环境依赖（仅开发 / 构建用，如 webpack、eslint、babel）npm install webpack --save-dev
peerDependencies对等依赖（要求宿主环境提供指定版本，如插件依赖 React 18）手动写进 package.json 或 npm install react --peer
optionalDependencies可选依赖（缺失不影响项目运行，如某些兼容插件）npm install xxx --save-optional

2. 版本号规则（关键：避免意外升级）

package.json 中的版本号有特殊符号，决定依赖安装的版本范围：

  • ^18.1.0：兼容更新（允许 18.x.x，不允许 19.0.0）→ 默认安装方式
  • ~18.1.0：补丁更新（允许 18.1.x，不允许 18.2.0）
  • 18.1.0：固定版本（仅安装 18.1.0）
  • *：安装最新版本（极度不推荐）

3. 锁定版本（核心：保证多人 / 多环境依赖一致）

  • package-lock.json：npm 自动生成，精确记录每个依赖的版本、下载地址、依赖树，npm install 时优先读取此文件，保证安装版本一致。
  • yarn.lock（yarn）/ pnpm-lock.yaml（pnpm）：功能同 package-lock.json，不同包管理器的锁定文件（不可混用）。

⚠️ 注意：

  • 必须将 lock 文件提交到 git 仓库，团队协作时同步；
  • 不要手动修改 lock 文件，由包管理器自动维护。

4. 依赖清理/更新

# 1. 清理未在 package.json 中声明的依赖（比如手动删除的包）
npm prune

# 2. 查看过时依赖
npm outdated

# 3. 更新指定依赖
npm update axios  # 更新到 ^ 允许的最新版本
npm install axios@latest  # 强制更新到最新版本

# 4. 全局更新 npm
npm install -g npm

# 5. 卸载依赖
npm uninstall axios  # 卸载生产依赖
npm uninstall webpack --save-dev  # 卸载开发依赖

5. 日常开发常用命令（核心）

# 1. 初始化项目（生成 package.json）
npm init  # 交互式创建
npm init -y  # 快速创建（默认配置）

# 2. 安装依赖（核心）
npm install  # 安装 package.json 中所有依赖（读 lock 文件）
npm install axios  # 安装生产依赖（加 --save 可省略）
npm install webpack --save-dev  # 安装开发依赖（简写 -D）
npm install axios@1.6.0  # 安装指定版本
npm install axios --force  # 强制安装（解决版本冲突）
npm install axios --legacy-peer-deps  # 忽略对等依赖冲突（老项目常用）

# 3. 运行脚本（package.json 中 scripts 字段）
npm run dev  # 运行开发脚本（如启动本地服务）
npm run build  # 运行构建脚本（如打包项目）
npm run test  # 运行测试脚本
npm start  # 简写：运行 start 脚本（无需 run）
npm stop  # 简写：运行 stop 脚本

# 4. 查看依赖信息
npm list  # 查看项目所有依赖树
npm list axios  # 查看指定依赖的版本
npm info axios  # 查看远程仓库中 axios 的信息（版本、描述等）
npm ls -g --depth=0  # 查看全局安装的依赖（仅显示一级）

# 5. 全局安装/卸载
npm install -g nrm  # 全局安装 nrm（npm 源管理工具）
npm uninstall -g nrm  # 全局卸载

# 6. 切换 npm 源（解决下载慢）
npm config set registry https://registry.npmmirror.com/  # 切换到淘宝源
npm config get registry  # 查看当前源
npm config delete registry  # 恢复默认源

进阶

  1. 使用 pnpm 替代 npm：pnpm 比 npm 更快、更节省磁盘空间，且依赖隔离更严格，命令基本兼容 npm（pnpm install/pnpm add/pnpm run）；
  2. 安装：npm install -g pnpm
  3. 用 nrm 管理 npm 源：快速切换淘宝源、官方源、公司私有源：
npm install -g nrm
nrm ls  # 查看所有源
nrm use taobao  # 切换到淘宝源
  1. 团队标准化：在项目根目录添加 .nvmrc/.node-version 文件，指定 Node 版本，团队成员用 nvm/volta 可自动识别：
# .nvmrc 文件内容
18.18.0
# 终端执行 nvm use 即可自动切换到该版本
